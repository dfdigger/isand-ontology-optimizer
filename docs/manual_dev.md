# Руководство программиста

Курсовой проект "Оптимизация работы с онтологией ISAND"

## 1. Назначение и условия применения программы

### 1.1. Назначение программы

Программа предназначена для построения модели онтологии, представленной в виде ациклического графа факторов, и сравнения двух методов подсчёта числа терминов на узел. Используется в учебных целях в рамках курсового проекта.

### 1.2. Функции, выполняемые программой

- разбор четырёх CSV-файлов и построение графа онтологии;
- структурная валидация графа (поиск циклов, узлов с несколькими родителями,
  узлов без уровня и имени);
- наивный и уникальный подсчёт числа терминов для каждого узла;
- сравнение результатов и выявление расхождений.

### 1.3. Условия применения

- операционная система: macOS, Linux или Windows;
- интерпретатор Python 3.10 или выше;
- минимальный объём оперативной памяти — 256 МБ;
- объём дискового пространства, достаточный для размещения исходных
  CSV-файлов в каталоге `data/raw/` (порядок десятков мегабайт).

## 2. Характеристика программы

### 2.1. Режимы работы

Программа работает в пакетном режиме: запускается из командной строки,
последовательно прогоняет модульные тесты и выполняет основной сценарий
(загрузка → валидация → подсчёты → сравнение).

### 2.2. Средства контроля правильности выполнения

- модульные тесты `src/tests/test_core.py` (модуль `unittest`);
- метод `OntologyGraph.validation_report()` — структурный отчёт о графе;
- сравнение результатов `NaiveCounter` и `UniqueCounter` в `src/main.py`.

При падении тестов программа завершается с кодом возврата `1` до выполнения
расчётов.

## 3. Обращение к программе

### 3.1. Запуск из командной строки

```bash
cd <корень репозитория>
python3 src/main.py
```

### 3.2. Запуск тестов отдельно

```bash
PYTHONPATH=src python3 -m unittest tests.test_core -v
```

### 3.3. Программный интерфейс

Импорт ключевых классов из пакета `core`:

```python
from core import OntologyGraph, NaiveCounter, UniqueCounter

graph = OntologyGraph.from_csv(
    factors_path="data/raw/factors.csv",
    edges_path="data/raw/factor_graph_edges.csv",
    roots_path="data/raw/factor_graph_roots.csv",
    variants_path="data/raw/factor_name_variants.csv",
)

report = graph.validation_report()
naive_counts  = NaiveCounter(graph).count_for_all_nodes()
unique_counts = UniqueCounter(graph).count_for_all_nodes()
```

## 4. Входные и выходные данные

### 4.1. Входные данные

CSV-файлы в каталоге `data/raw/`:

| Файл                          | Поля                                              |
|-------------------------------|---------------------------------------------------|
| `factors.csv`                 | `id`, `level`                                     |
| `factor_graph_edges.csv`      | `id`, `predecessor_id`, `successor_id`            |
| `factor_graph_roots.csv`      | `id`, `root_id`                                   |
| `factor_name_variants.csv`    | `id`, `factor_id`, `variant`, `language_id`       |

Если узел встречается в `factor_graph_edges.csv`, но отсутствует в
`factors.csv`, ему присваивается уровень `-1` и имя `factor_<id>`.
Если файл корней не задан или пуст, корнями считаются узлы без родителей.

### 4.2. Выходные данные

Стандартный поток вывода:

- секция `=== validation_report ===` — словарь со сводкой по графу;
- секции `=== naive: ===` и `=== unique: ===` — фрагменты словарей
  «идентификатор узла → значение счётчика»;
- секция `=== узлы, где naive != unique: N ===` — список троек
  `(nid, naive, unique)`.

## 5. Описание логической структуры

### 5.1. Состав модулей

| Модуль                                    | Назначение                                    |
|-------------------------------------------|-----------------------------------------------|
| `src/main.py`                             | точка входа; запуск тестов и расчётов         |
| `src/core/ontology_graph.py`              | модель онтологии, чтение CSV, валидация       |
| `src/core/naive_counter.py`               | наивный счётчик с мемоизацией                 |
| `src/core/unique_counter.py`              | уникальный счётчик с мемоизацией множеств     |
| `src/core/__init__.py`                    | реэкспорт классов из пакета `core`            |
| `src/tests/test_core.py`                  | модульные тесты на ручных графах              |

### 5.2. Класс `OntologyGraph`

Хранит:

- `level_by_id: Dict[int, int]` — уровень узла;
- `name_by_id: Dict[int, str]` — отображаемое имя;
- `children_by_id: Dict[int, Set[int]]` — список потомков;
- `parents_by_id: Dict[int, Set[int]]` — список родителей;
- `roots: Set[int]` — корни графа.

Ключевые методы:

- `from_csv(...)` — фабричный конструктор из четырёх CSV;
- `nodes()`, `children()`, `parents()`, `is_leaf()`, `leaves()`;
- `multi_parent_nodes()` — узлы с несколькими родителями;
- `detect_cycle()` — поиск цикла классическим DFS с раскраской вершин;
- `validation_report()` — сводка о графе.

### 5.3. Класс `NaiveCounter`

- Поле `memo: Dict[int, float]` — кэш по идентификатору узла.
- Метод `count_for_node(node_id, term_weight=None)` — рекурсивно считает
  сумму вкладов по всем путям до листьев. Если `term_weight=None`, каждый
  лист даёт `1.0`, иначе — значение `term_weight(leaf_id)`.
- Метод `count_for_all_nodes(...)` в начале очищает кэш и считает значения
  для всех узлов.
- Метод `reset_cache()` — ручная очистка кэша.

### 5.4. Класс `UniqueCounter`

- Поле `memo_terms: Dict[int, Set[int]]` — кэш множеств уникальных листьев.
- Метод `terms_for_node(node_id)` — множество идентификаторов уникальных
  листьев, достижимых из узла; для листа возвращает `{node_id}`.
- Метод `count_for_node(node_id, term_weight=None)` — размер множества или
  сумма весов уникальных листьев.
- Метод `count_for_all_nodes(...)` — аналогично наивному, с очисткой кэша.

### 5.5. Алгоритмическая сложность

Пусть `N` — число узлов, `E` — число рёбер, `L` — суммарная мощность
множеств уникальных листьев по всем узлам.

- `OntologyGraph.from_csv` — `O(N + E)`.
- `OntologyGraph.detect_cycle` — `O(N + E)`.
- `NaiveCounter.count_for_all_nodes` — `O(N + E)` при использовании кэша,
  поскольку каждый узел рассчитывается единожды.
- `UniqueCounter.count_for_all_nodes` — `O(L)` дополнительной работы на
  объединения множеств.

### 5.6. Различие в результатах

Для узла, у которого один и тот же лист достижим по нескольким путям,
`NaiveCounter` суммирует вклад этого листа столько раз, сколько различных
путей; `UniqueCounter` — один раз, на этом и основан смысл сравнения двух
методов.

## 6. Тестирование

### 6.1. Состав тестов

В файле `src/tests/test_core.py` реализованы тесты:

- `test_chain_counts_match` — цепочка `1→2→3`;
- `test_triangle_counts_match` — корень и два листа;
- `test_diamond_root_naive_doubles_unique` — ромб, расхождение в корне;
- `test_diamond_inner_nodes_agree` — для внутренних узлов ромба совпадение;
- `test_count_for_all_nodes_same_keys` — совпадение множеств ключей;
- `test_unique_with_weight_matches_manual` — учёт весов в уникальном счётчике;
- `test_naive_with_weight_diamond_doubles_unique` — учёт весов в наивном
  счётчике при наличии нескольких путей.

### 6.2. Запуск

Тесты автоматически выполняются перед основным сценарием в `src/main.py`.
Изолированный запуск:

```bash
PYTHONPATH=src python3 -m unittest tests.test_core -v
```

## 7. Сообщения программисту

- `ModuleNotFoundError: No module named 'core'` — каталог `src` не добавлен в
  `PYTHONPATH`; запускать через `python3 src/main.py` либо явно задавать
  переменную окружения `PYTHONPATH=src`.
- `FileNotFoundError: ... data/raw/factors.csv` — отсутствует один из
  CSV-файлов в каталоге `data/raw/`.
- `KeyError: 'predecessor_id'` или аналогично — повреждён заголовок CSV-файла.
- `RecursionError` — превышена глубина рекурсии при расчётах на графе с
  очень глубокими цепочками; следует увеличить лимит вызовом
  `sys.setrecursionlimit(...)` либо переписать алгоритм в итеративной форме.
