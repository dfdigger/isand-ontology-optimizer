# Руководство оператора

Курсовой проект "Оптимизация работы с онтологией ISAND"

## 1. Назначение программы

Программа предназначена для построения модели онтологии и сравнения двух методов подсчёта числа терминов на узел: наивного и уникального. Результаты выводятся в стандартный поток вывода (консоль). Программа рассчитана на изолированную оптимизацию и не требует взаимодействия с базами данных или сетевыми сервисами.

## 2. Условия выполнения программы

### 2.1. Минимальные требования

- персональный компьютер с операционной системой macOS, Linux или Windows;
- установленный интерпретатор Python 3.10 или выше;
- не менее 256 МБ оперативной памяти;
- свободное дисковое пространство для размещения исходных CSV-файлов
  (порядок десятков мегабайт).

### 2.2. Подготовка рабочего места

1. Получить копию проекта (например, командой `git clone <url>` либо
   распаковкой архива в произвольный каталог).
2. Открыть терминал и перейти в корневой каталог проекта.
3. Создать виртуальное окружение и активировать его:

   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

4. Поместить в каталог `data/raw/` четыре CSV-файла:

   - `factors.csv`;
   - `factor_graph_edges.csv`;
   - `factor_graph_roots.csv`;
   - `factor_name_variants.csv`.

   Файлы должны быть в кодировке UTF-8 и содержать заголовки столбцов.

## 3. Выполнение программы

### 3.1. Запуск

Из корневого каталога проекта выполнить команду:

```bash
python3 src/main.py
```

### 3.2. Сценарий работы

После запуска программа последовательно выполнит следующие действия.

1. Прогон встроенных модульных тестов. На экране появится список тестов с
   отметкой `ok`. Если хотя бы один тест завершится неуспешно, программа
   остановится с кодом возврата `1`, а основные расчёты выполнены не будут.
2. Загрузка онтологии из CSV-файлов в каталоге `data/raw/`.
3. Печать отчёта о валидации графа (заголовок `=== validation_report ===`).
4. Печать первых 40 значений наивного и уникального счётчиков
   (заголовки `=== naive: ===` и `=== unique: ===`).
5. Печать количества и перечня узлов, для которых результаты двух
   методов различаются (заголовок `=== узлы, где naive != unique: N ===`).

## 4. Сообщения оператору

Возможные сообщения и реакция на них.

| Сообщение                                          | Значение и действия оператора                                                                                                  |
|----------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------|
| `Ran N tests in ...s OK`                           | Все тесты пройдены, можно продолжать наблюдение за выводом.                                                                   |
| `FAILED (failures=...)` и завершение с кодом `1`   | Один или несколько тестов не прошли. Расчёты не выполняются. Проверить целостность исходного кода и ещё раз запустить тесты. |
| `FileNotFoundError: ... data/raw/<имя>.csv`        | Отсутствует один из CSV-файлов. Поместить файл в каталог `data/raw/` и повторить запуск.                                       |
| `KeyError: 'predecessor_id'` и аналогичные         | В CSV-файле отсутствует ожидаемый заголовок столбца. Проверить структуру файла согласно разделу 2.2.                          |
| `=== узлы, где naive != unique: 0 ===`             | Различий между методами нет (граф является деревом или не содержит общих потомков). Это нормальная ситуация.                  |
| `=== узлы, где naive != unique: N ===` при `N > 0` | Найдено `N` узлов, в которых наивный счётчик завышает значение по сравнению с уникальным. Список выводится сразу после строки. |

## 5. Завершение работы

Программа завершает работу автоматически после печати последней секции.
Для деактивации виртуального окружения выполнить:

```bash
deactivate
```

## 6. Контрольный пример

Корректность установки и запуска можно проверить без реальных данных,
выполнив только модульные тесты:

```bash
PYTHONPATH=src python3 -m unittest tests.test_core -v
```

В случае успеха в конце вывода появится строка `OK`, а программа вернёт
код возврата `0`.
