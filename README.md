# isand-ontology-optimizer

Курсовой проект "Оптимизация работы с онтологией ISAND"

## 1. Общие сведения

- **Наименование программы:** `isand-ontology-optimizer`.
- **Назначение:** Валидация и улучшение методов подсчёта терминов в иерархии онтологии.
- **Язык программирования:** Python 3.13
- **Стандартные библиотеки:** `csv`, `dataclasses`, `pathlib`, `typing`,
  `unittest`, `pprint`.
- **Тип программы:** консольное приложение, запускаемое одним сценарием
  `src/main.py`.

## 2. Функциональное назначение

Программа выполняет следующие функции:

1. Загружает онтологию из четырёх CSV-файлов (данные онтологии)
   (`data/raw/factors.csv`, `data/raw/factor_graph_edges.csv`,
   `data/raw/factor_graph_roots.csv`, `data/raw/factor_name_variants.csv`)
   и строит объект `OntologyGraph`.
2. Проводит структурную валидацию графа: подсчёт узлов, рёбер, корней, листьев,
   обнаружение циклов, выявление узлов с несколькими родителями, узлов без
   уровня и без имени.
3. Выполняет подсчёт числа терминов для каждого узла двумя методами:
   - наивным (`NaiveCounter`) — суммирование вкладов по всем путям до листьев;
   - уникальным (`UniqueCounter`) — суммирование вкладов по множеству
     достижимых уникальных листьев.
4. Сравнивает результаты двух методов и формирует список узлов, для которых
   результаты различаются.
5. Перед расчётами автоматически прогоняет модульные тесты из
   `src/tests/test_core.py`, подтверждающие корректность реализаций на
   эталонных графах (цепочка, треугольник, ромб) и наличие расхождений в
   ожидаемых случаях.

## 3. Используемые технические средства

- Персональный компьютер с операционной системой macOS, Linux или Windows.
- Установленный интерпретатор Python версии 3.10 или выше.
- Для хранения исходных данных — каталог `data/raw/` в корне проекта.

## 5. Входные и выходные данные

### 5.1. Входные данные

Все исходные файлы — CSV с заголовком:

| Файл                          | Поля                                              |
|-------------------------------|---------------------------------------------------|
| `factors.csv`                 | `id`, `level`                                     |
| `factor_graph_edges.csv`      | `id`, `predecessor_id`, `successor_id`            |
| `factor_graph_roots.csv`      | `id`, `root_id`                                   |
| `factor_name_variants.csv`    | `id`, `factor_id`, `variant`, `language_id`       |

### 5.2. Выходные данные

Результаты выводятся в стандартный поток вывода:

- отчёт о валидации графа (`validation_report`);
- словари "id узла: значение для этого id" для двух методов
  (для краткости печатаются первые 40 узлов);
- перечень узлов, для которых наивный и уникальный счётчики дали разные
  значения.

## 6. Установка и запуск

### 6.1. Подготовка окружения

```bash
git clone https://github.com/dfdigger/isand-ontology-optimizer.git
cd isand-ontology-optimizer
python3 -m venv venv
source venv/bin/activate
pip install -U pip
```

### 6.2. Размещение данных

Поместить четыре CSV-файла из раздела 5.1 в каталог `data/raw/` с указанными
именами.

### 6.3. Запуск

```bash
python3 src/main.py
```

Сценарий выполнит модульные тесты, после чего распечатает отчёт о валидации
и результаты подсчётов. При падении тестов программа завершится с кодом
возврата `1` и расчёты не будут выполнены.
